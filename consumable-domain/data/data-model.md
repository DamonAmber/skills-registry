# 耗材数据定义

本文件说明耗材余量与状态的数据逻辑。

## 一、耗材状态定义

耗材余量依赖于关联的 Spec 属性（Property），厂商在开发者平台通过设置临界值定义三个状态：

| 状态 | 说明 | 配置 |
|------|------|------|
| 充足 | 正常使用，无需更换 | 无需配置，非不足/耗尽即为充足 |
| 不足 | 需关注，建议尽快更换 | 临界值厂商必填 |
| 耗尽 | 已耗尽，必须更换 | 临界值厂商选填 |

## 二、Spec 属性值类型与状态定义方式

| 类型 | 示例耗材 | 状态定义方式 | 备注 |
|------|---------|-------------|------|
| 数值列表 | 软水盐（洗碗机） | 枚举值映射，如 Low=不足 | 主流方式 |
| 数值范围 | 电池（人体传感器） | 阈值百分比，如 ≤5%=不足，≤3%=耗尽 | 主流方式 |
| 布尔值 | 电池（宠物喂食器） | TRUE=不足，其他=充足 | 不建议，仅约 10 个 Model 使用，分析为错误使用 |

## 三、提示策略与消息推送

- 耗材状态由 Spec 属性值驱动，属性变化触发状态更新
- 消息推送支持标准消息（Admin 平台模板）和自定义消息（厂商配置）
- 推送配置项：消息名称、触发条件（属性/事件）、对比条件、推送范围、用户可关闭、推送最小间隔
- 标准消息优先级高于自定义消息（触发条件一致时阻断自定义消息）

## 四、支持查询

耗材支持标准查询服务，供语控、米家助手等使用：

- 设备信息（名称、PID、Model、did）
- 耗材名称
- 耗材余量（数值范围类型）
- 耗材状态（充足、不足、耗尽）
- 耗材型号
- 耗材建议更换周期
- 耗材购买链接
