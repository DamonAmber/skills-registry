# 耗材组合购买设计决策

本文件提炼自 2026-03 耗材组合购买 PRD 和 UI 方案评审，记录已确认的设计原则和方案选择。

## 一、业务背景

小米商城希望在米家 App 耗材页面增加"组合购买"入口，推广耗材卡权益。

核心约束：
- 无法从商城获取具体优惠金额，只知道"耗材卡可享折扣"
- 弹窗内购买流程由商城 SDK 承接
- 需要在三个耗材页面分别出方案

## 二、设计原则

1. **设备耗材页的主职责是展示耗材余量与维护状态**，不是销售或购物车页面
2. 组合购买只能作为上级页里的**弱提示入口**
3. 价格与组合选择由小米商城 SDK 页面承接，**米家侧不展示具体价格**
4. 不做购物车心智
5. 保留每个耗材项的单买路径

## 三、三个页面的推荐方案

### 3.1 首页耗材弹窗 → 独立组合购买卡片

- 在耗材卡片列表与底部按钮之间插入独立卡片
- 品牌绿色浅底，与耗材卡片有视觉差异
- 用户心智：扫描式浏览，购买意向弱，适合弱提示

### 3.2 设备耗材页 → 耗材不足分组内嵌入口

- 在"耗材不足"分组末尾嵌入组合购买入口行
- 时机最精准：用户刚看完哪些耗材需要买
- 用户心智：管理式浏览，有一定购买意向

### 3.3 耗材详情页 → 底部双按钮

- 主按钮保持"立即购买"，下方增加次要按钮"组合购买"
- 用户心智：聚焦式决策，购买意向最强
- 推荐方案 D3（组合购买整体模块强化），备选 D2

## 四、三个页面的递进关系

弹窗"发现" → 耗材页"了解" → 详情页"决策"

| 页面 | 适合程度 | 角色 |
|------|---------|------|
| 耗材详情页 | 最适合 | 核心转化场景 |
| 设备耗材页 | 适合 | 发现入口 |
| 首页耗材弹窗 | 可以但优先级低 | 补充触点 |

## 五、折扣信息展示策略

统一使用"享耗材卡折扣"模糊表述，不展示具体折扣数字。原因：
- 不承诺具体金额，避免与商城侧数据不一致
- 具体折扣信息在商城 SDK 弹窗中展示，由商城侧负责准确性
