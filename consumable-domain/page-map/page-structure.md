# 耗材页面结构与导航

本文件描述耗材在米家 App 中的页面层级、入口和展示形态。

## 一、三个展示入口

| 入口 | 位置 | 展示内容 | 触发条件 |
|------|------|---------|---------|
| 首页仪表盘 | 首页（米家）Tab | 仅不足/耗尽状态耗材 | 有耗材不足时才展示 |
| 设备耗材页 | 我的 → 设备耗材 | 全量耗材列表 | 始终可进入 |
| 设备插件 | 设备详情内 | 耗材卡片 | 设备有耗材时展示 |

## 二、页面层级

```
首页仪表盘 → 耗材不足半屏抽屉弹窗 → 耗材详情页
我的 → 设备耗材（列表）→ 耗材详情页
设备插件 → 耗材卡片 → 耗材详情页
```

三条路径最终都汇聚到**耗材详情页**。

## 三、首页仪表盘耗材弹窗

- 弹窗类型：半屏抽屉弹窗（从底部上滑）
- 触发条件：有耗材处于不足/耗尽状态时，仪表盘展示耗材入口
- 展示内容：不足/耗尽状态的耗材列表
- 用户心智："我家哪些耗材需要关注？"——扫描式浏览
- 点击列表项跳转耗材详情页

## 四、设备耗材页

- 集中展示全家庭耗材状态
- 按状态分组：耗材不足 / 状态未知 / 充足
- 每条显示：剩余百分比/状态、耗材类型、设备名称
- 用户心智："我要看看所有设备的耗材情况"——管理式浏览

## 五、耗材详情页

耗材详情页根据配置有多种形态：

| 形态 | 触发条件 | 额外元素 |
|------|---------|---------|
| 默认 | 有余量百分比 | 耗材名称、剩余百分比、设备图、更新时间 |
| 仅重置 | 支持重置 | + 重置按钮 |
| 仅购买 | 配置购买链接 | + 「立即购买」按钮 |
| 更换教学 | 配置教学内容 | + 教学入口（视频/图文） |
| 建议更换时间 | 配置更换周期 | + 预估使用天数 |
| 重置+购买 | 同时支持 | + 重置按钮 + 购买按钮 |

通用元素：设备图、耗材名、耗材状态、更新时间、耗材型号、功能介绍。

## 六、插件耗材卡片

- 历史现状：部分插件自行实现耗材卡片，展示逻辑不统一
- 标准化方案：客户端提供 Native 通用耗材详情页，插件通过传参调起
- 倾向方向：标准化使用 Native 通用页面

## 七、展示样式差异

| 属性值类型 | 展示方式 | 示例 |
|-----------|---------|------|
| 数值范围型 | 进度条 + 百分比 | 电池 ≤5% |
| 数值列表型 | 状态文字（充足/不足/耗尽） | 软水盐 Low |
